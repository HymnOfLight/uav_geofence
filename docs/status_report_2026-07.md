# 电子围栏 INT8 QNN 验证：项目现状、领域定位与推进方向

**组内讨论稿 · 2026 年 7 月 21 日**

---

## 0. 一段话总结

我们的流水线（教师 → FP32 MLP → INT8 QNN → E0 语义一致性 → E1 单步 SMT 包络 → E2 闭环可达性 → 蒙特卡洛 + 屏蔽）已经从纯合成数据推进到了**真实开源飞行数据 + 真实固件（SITL）**：logs.px4.io 的公开飞行日志可以两条命令进训练，ArduCopter 真实固件的绕栏飞行数据已经端到端采集并通过全流程验证。按「合成数据 → 开源平台 → 商用真机」的阶梯，我们在第 2 级（开源平台）完成了 SITL 一半，**下一格是开源平台真机飞行，工具链已经就位**。领域里没有人打通「位精确 QNN 验证 + 围栏场景 + 固件在环证据」这个组合，这是我们的差异化位置；但往大疆等商用平台走，架构上只有「外挂监督层（RTA）」一条路，且模型一致性检验（conformance）是最大的方法学缺口。

## 1. 项目现状：五级阶梯，我们在哪

| 阶段 | 状态 | 说明 |
|---|---|---|
| ① 合成数据 + 自训练网络 | 【已完成】 | E0 100% 一致；E1/E2/MC 全流程可复现，五种子扫描脚本就位 |
| ② 真实开源飞行数据 | 【已完成】 | logs.px4.io 公开日志自动抓取/过滤/校验；ULog/DataFlash/tlog/CSV 四种源；auto_align 自动对齐围栏几何 |
| ③ 开源固件在环（SITL） | 【已完成】 | ArduCopter V4.8.0-dev 真实固件：无头全自动飞行（`sitl-fly`）+ 并行录制，绕栏数据过全流程 |
| ④ 开源平台真机 | 【未开始】 | 缺硬件；软件侧只需把 MAVLink URL 从 SITL 换成真机电台/伴飞计算机 |
| ⑤ 商用平台（大疆等） | 【未开始】 | 结构性依赖外挂监督层架构，见第 3 节 |

**阶段 ②③ 的关键实测数字**（都可从仓库一键复现）：

- 真实公开日志：3 条真实四旋翼飞行（410–657 s）进入训练，全流程约 7 分钟跑完；E0 200/200 一致，E1 200 单元产出 60 SAFE / 69 候选 UNSAFE / 71 UNKNOWN，蒙特卡洛 5 组控制器 0 穿越；
- 固件在环：SITL 任务从 (-35,0) 绕过 x∈[-5,5] 的禁飞矩形到达 (70,0)；录得 5608 样本，**禁飞盒内样本 0 个，最小间距 6.98 m**，峰值速度 7.87 m/s 贴住 WP_SPD=8 限速——固件（BendyRuler + 多边形围栏）真实绕行而非触发违规动作；
- 工程副产品：PX4/ArduPilot 行为教师（参数名与固件一一对应）、从实验配置自动生成 QGC 围栏计划与两代固件参数文件的生成器、root 容器/无显示环境的完整绕过路径（组里在 AutoDL 上直接可用）。

**当前的诚实边界**：被控对象仍是 2D 双积分器；E2 是保守区间可达性原型而非 Flow* 级后端；行为教师是文档级建模不是固件本体（固件数据走日志/SITL 路径）；真实航迹不会绕我们的虚拟围栏飞，训练时按 0.5 比例混入教师样本补避障行为——论文里这条必须写清楚。

## 2. 领域里其他人做到了什么

| 工作 | 核心方法 | 到达阶段 |
|---|---|---|
| NASA Safeguard（2016–2019 DASC 系列） | 独立硬件安全网；边界检测/违规预测算法 PVS 形式化验证 | **最成熟**：多机型真机大量飞行测试（含 GPS 退化、BVLOS、UTM 集成），已进入技术转让 |
| NASA ICAROUS + PolyCARP（开源） | PVS 验证的多边形围栏库 + 伴飞计算机架构，MAVLink 对接 | 开源平台真机（ArduCopter 飞行测试）；PolyCARP 有浮点实现级证明（FM 2019） |
| NASA reference governor（2019） | 考虑动力学与不确定性的速度约束 + 指令过滤，PVS 证明 | 仿真/合成阶段 |
| CBF 安全滤波系列（NTNU/ETH/Stanford，2024–25 活跃） | 控制屏障函数做加速度级安全滤波，最新工作嵌入 PX4 固件（开源 fork） | 开源平台真机（室内外、含对抗指令）；但证明停留在连续动力学层，不覆盖实现/量化 |
| HJ 可达性 / FaSTrack（Berkeley 等） | Hamilton–Jacobi 可达集 | 仿真 + 动捕房间小平台 |
| ACAS Xu NN 验证（Marabou/nnenum/NNV/α,β-CROWN，VNN-COMP） | 开环性质验证 → 闭环 NNCS 可达性 | 合成/基准阶段；闭环全验证公认超出现有工具；网络未上机审定 |
| QNN 验证（QNNVerifier、QEBVerif 等） | 量化网络位精确 SMT 编码 | 合成/基准阶段（与我们的 E0/E1 同层） |
| RTA 标准化（ASTM F3269；auto-GCAS 系谱） | Simplex/运行时保障架构标准 | 标准已发布修订中；auto-GCAS 有大量真机 |
| 工业部署（DJI GEO、PX4/ArduPilot 原生围栏） | 工程实现，无公开形式化保证 | 部署规模最大、保证等级最低；**DJI 自 2025-01 起在美国退为可跳过的警告**（欧洲更早） |

**三条格局判断**：

1. 形式化做得最深的（Safeguard/PolyCARP）验证的是**传统几何算法**，不是神经网络；
2. 神经网络/量化网络验证做得最深的（ACAS Xu、QNN 工具）停在**基准阶段**，没有上机；
3. CBF 上了真机但不覆盖实现层。→ 「位精确 QNN + 围栏场景 + 固件在环证据链」这个组合当前没有人打满阶梯，是我们的空位。

## 3. 往大疆等商用平台走，缺什么

1. **架构**：大疆固件闭源，唯一现实形态是**外挂监督层（RTA/Simplex）**——伴飞计算机经 MSDK/PSDK 虚拟摇杆发速度指令（接口限幅水平 ±15 m/s、垂直 ±4 m/s），把大疆飞控当黑箱。注意控制权级联（遥控器 > MSDK > OSDK），安全论证必须覆盖「监督层被人工旁路」。
2. **模型一致性（最大方法学缺口）**：「虚拟摇杆指令 → 真机响应」本身是带滞后限幅的内环闭环，需要黑箱系统辨识 + **conformance testing**（用真实飞行数据证明验证模型的扰动界过近似真机），否则 E1/E2 的 SAFE 对真机不成立。切入点：大疆日志（DJI Fly txt / 机内 DAT）可用 Airdata/DatCon 解码成 CSV，直接进我们现有的 `csv` 数据链路。
3. **几何与定位**：平面系 → WGS84、任意多边形、高度基准（AGL/MSL/椭球高）、动态禁飞区；GNSS 退化/欺骗下的完整性界——Safeguard 专门测过 GPS 退化，这是学术原型普遍缺的一层。
4. **验证规模**：3D + 更大网络会让 E1/E2 爆炸，需要接 α,β-CROWN / nnenum / Flow* 级后端；INT8 语义要对齐真实 NPU；屏蔽器自身浮点实现也要验证（PolyCARP 的 FM 2019 是标杆）。
5. **认证路径**：ASTM F3269（RTA）+ DO-178C/DO-333（形式化方法补充）+ 成形中的 ED-324/ARP6983（AI 审定）；欧盟侧 EASA 等级标志的 geo-awareness 要求。我们的 UNKNOWN 不计安全、逐单元 CSV/JSON、模型哈希溯源在方向上已对齐这套证据学。
6. **时局**：大疆取消强制围栏（合规责任移交飞手 + Remote ID 执法）意味着「机内围栏」在退潮，**「可证明的独立第三方围栏执行层」**（运营方/监管方加装）反而是更清晰的定位——建议论文叙事直接采用这个框架。

## 4. 建议的推进路线与讨论问题

**短期（现有硬件预算内）**：树莓派/Jetson + Pixhawk 真机复现 SITL 流程（软件已就位，改 URL 即可），把证据推到「开源平台真机」，对齐 ICAROUS 的部署形态；同时补 E2 更强后端的接口实验。

**中期**：解码大疆日志进 `csv` 链路（「大疆数据」阶段）；3D 化；把 conformance testing 做成方法学贡献（这块领域里没有标准做法，可能是独立的一篇）。

**长期**：以 ASTM F3269 架构把验证过的 INT8 QNN + 屏蔽做成 PSDK 外挂在大疆机上飞（「大疆平台」阶段）；学术贡献主要在一致性论证与证据链方法学，不在控制本身。
